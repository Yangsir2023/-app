# 🐾 宠物领养平台 - 启动与配置指南

本项目已完成前后端一体化改造，前端使用 React + Vite，后端使用 Express + Supabase。以下是启动项目的详细步骤。

## 1. 准备工作：Supabase 控制台配置

由于使用了 Supabase 数据库，您需要执行以下操作：

1.  访问 [Supabase 官网](https://supabase.com/) 并创建一个新项目。
2.  **设置数据库结构**：
    *   在 Supabase 控制台打开 **SQL Editor**。
    *   将项目根目录下的 `supabase/schema.sql` 文件内容粘贴进去并运行。这将创建 `pets`, `rescue_stations`, `faqs`, `adoption_applications`, 和 `favorites` 表。
3.  **获取 API 密钥**：
    *   前往 **Project Settings** -> **API**。
    *   复制 **Project URL**。
    *   复制 **anon public key**。
    *   复制 **service_role secret**（注意：这通常在下方的 "Service Role" 部分，需要点击揭晓）。

## 2. 配置环境变量

在项目根目录下创建一个名为 `.env` 的文件（如果不存在），并将获取到的密钥填入：

```env
# Supabase 配置
SUPABASE_URL="您的_PROJECT_URL"
SUPABASE_SERVICE_ROLE_KEY="您的_SERVICE_ROLE_KEY"

# 前端环境变量
VITE_SUPABASE_URL="您的_PROJECT_URL"
VITE_SUPABASE_ANON_KEY="您的_ANON_PUBLIC_KEY"

# 后端端口
SERVER_PORT=3001
```

## 3. 快速启动步骤

请在终端中按顺序执行以下命令：

### 第一步：安装依赖
```bash
npm install
```

### 第二步：导入初始数据 (Seed)
确保 `.env` 已正确配置后，运行以下命令将宠物信息导入数据库：
```bash
npm run seed
```

### 第三步：启动前后端
使用复合命令同时启动 Vite 前端和 Express 后端：
```bash
npm run dev:all
```
*   前端地址：`http://localhost:3000`
*   后端接口：`http://localhost:3001`

## 4. 常见问题排查 (解决 npm run dev 失败)

如果您运行 `npm run dev` 或 `npm run dev:all` 报错，请检查以下几点：

### ❌ 报错：`command not found: concurrently` 或 `tsx`
**原因**：依赖项未正确安装。
**解决**：运行 `npm install`。本项目依赖 `concurrently` 来同时跑两个服务，依赖 `tsx` 来运行后端 TypeScript 代码。

### ❌ 报错：`Error: ❌ 缺少 Supabase 配置！`
**原因**：`.env` 文件缺失或变量名写错。
**解决**：确保根目录下有 `.env` 文件，且变量名与 `server/lib/supabase.ts` 中读取的一致。

### ❌ 报错：`Port 3000 is already in use`
**原因**：端口被占用。
**解决**：关闭其他占用 3000 或 3001 端口的程序，或者在 `package.json` 中修改端口号。

### ❌ 报错：`Could not resolve "@tailwindcss/vite"`
**原因**：使用了 Tailwind v4 的 Vite 插件但未安装或环境不匹配。
**解决**：确保 `npm install` 已执行。本项目使用的是最新版本的 Tailwind。

## 5. 开发建议
*   **前端逻辑**：可见 `src/lib/api.ts`，所有数据请求都封装在此。
*   **后端逻辑**：可见 `server/routes/`，按模块划分了宠物、救助站、申请等路由。
*   **数据持久化**：所有的收藏记录和领养申请现在都会真实保存到 Supabase 数据库中。
